build - Intelligent project implementation from documentation

Usage: Automatically detect and implement projects from opus documentation suites using contextual intelligence and specification analysis.

**Intelligent Documentation-Driven Build Protocol:**

🎯 **Phase 1: Context Discovery & Documentation Analysis**
- Auto-detect opus documentation suite in current directory (README.md, PURPOSE.md, INTENT.md, CONCEPTS.md, METHODS.md, CLAUDE.md, SPECIFICATION.md)
- Parse project specifications, technical requirements, and architectural patterns
- Extract programming language, frameworks, and dependency requirements from SPECIFICATION.md
- Analyze PURPOSE.md for core objectives and INTENT.md for user goals
- Validate documentation completeness and cross-file consistency

🔍 **Phase 2: Project Intelligence & Technology Selection**
- Determine optimal project structure from CONCEPTS.md architecture principles
- Select build tools, package managers, and development environment from METHODS.md
- Configure testing frameworks and quality assurance requirements
- Plan implementation strategy based on complexity and feature requirements
- Generate development roadmap with milestone validation checkpoints

🏗️ **Phase 3: Foundation & Structure Implementation**
- Create project directory structure per SPECIFICATION.md architecture
- Generate configuration files (package.json, Cargo.toml, requirements.txt, etc.)
- Initialize version control with appropriate .gitignore and repository setup
- Set up development environment, dependencies, and build systems
- Create base application structure and entry points

⚙️ **Phase 4: Core Feature Implementation**
- Implement API endpoints and core functionality from SPECIFICATION.md
- Build user interfaces aligned with INTENT.md user experience requirements
- Add business logic following PURPOSE.md objectives and value propositions
- Implement data models, validation, and error handling systems
- Create CLI interfaces and command-line tools if specified

🔧 **Phase 5: Quality Assurance & Testing Integration**
- Generate comprehensive test suites based on METHODS.md testing strategies
- Implement security measures and input validation protocols
- Add monitoring, logging, and observability features
- Create deployment scripts and automation following CLAUDE.md agent guidelines
- Validate all features against original documentation specifications

✅ **Phase 6: Documentation Synchronization & Deployment Preparation**
- Update implementation documentation to match actual code
- Generate API documentation and usage examples
- Create deployment guides and operational procedures
- Verify project completeness against opus documentation requirements
- Prepare production-ready builds and distribution packages

**Smart Context Detection:**

🔍 **Automatic Documentation Discovery:**
- Detects opus 7-file documentation suites automatically
- Falls back to README.md analysis for basic project understanding
- Analyzes existing code structure for enhancement vs. new project creation
- Identifies project type, language, and framework from documentation

📋 **Intelligent Requirement Extraction:**
- Parses SPECIFICATION.md for technical architecture and API definitions
- Extracts user stories and success criteria from INTENT.md
- Analyzes CONCEPTS.md for domain terminology and architectural patterns
- Uses METHODS.md for implementation approach and testing strategies

**Implementation Examples:**

```bash
# Auto-build from opus documentation (no arguments needed)
build

# Build with specific strategy override
build --strategy minimal

# Build with verbose progress reporting
build --verbose --report

# Build with custom technology stack
build --language typescript --framework react
```
