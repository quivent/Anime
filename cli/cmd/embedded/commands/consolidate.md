consolidate - Intelligent fragment unification into coherent wholes

Usage: Merge multiple scattered fragments, projects, or utilities into a unified, coherent system with consistent interfaces and deduplicated logic.

**Sequential Consolidation Protocol:**

🔍 **Phase 1: Source Discovery & Inventory**
- Enumerate all input sources (paths, projects, files)
- Generate content inventory with metadata and provenance
- Identify overlapping functionality and potential duplicates
- Map dependencies within and across sources
- Create source fingerprints for traceability

🎯 **Phase 2: Unification Strategy Design**
- Synthesize approach from perspectives:
  - **Librarian**: Optimal categorization and organization
  - **Diplomat**: Fair conflict resolution preserving all value
  - **Editor**: Coherent narrative from disparate pieces
- Define target unified structure
- Create merge precedence rules and conflict policies

🔬 **Phase 3: Duplicate Analysis & Resolution**
- Detect exact duplicates via content hashing
- Identify near-duplicates through similarity analysis
- Categorize: Identical, Evolved, Forked, Coincidental
- Define resolution strategy per category
- Preserve provenance and attribution

🔧 **Phase 4: Merge Execution**
- Execute consolidation in dependency-safe order:
  - **Deduplicate**: Eliminate redundant content
  - **Merge**: Combine complementary implementations
  - **Harmonize**: Align interfaces and conventions
  - **Unify**: Create single coherent structure
- Track all merge decisions for audit

✅ **Phase 5: Coherence Validation**
- Verify unified output contains all source functionality
- Validate no unresolved conflicts remain
- Generate coverage report (what came from where)

**Integration Patterns:**

```bash
# Consolidate three utilities into one
consolidate ./utils-a ./utils-b ./utils-c --output ./unified --strategy weave

# Merge with conflict review
consolidate ./docs-old ./docs-new --output ./docs --interactive

# Preview consolidation plan
consolidate ./project-* --dry-run --provenance
```

**Merge Strategies:**
- `absorb`: One source becomes base, others merge in
- `weave`: Interleave sources by category/domain
- `extract`: Pull common elements into shared core
- `synthesize`: Create new structure from source elements

**Conflict Policies:**
- `newest`: Most recently modified wins
- `largest`: Most comprehensive kept
- `manual`: Human decision required
- `preserve`: Keep both as variants

**Quality Standards:**
- ✅ **Completeness**: All source functionality present
- ✅ **Coherence**: Unified structure with consistent interfaces
- ✅ **Deduplication**: No unnecessary redundancy
- ✅ **Provenance**: Clear attribution of all origins

Target: $ARGUMENTS

The consolidate command delivers intelligent unification guided by librarian organization and diplomatic conflict resolution, ensuring nothing valuable is lost while everything becomes findable and unified.
