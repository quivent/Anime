# construct Command - Dynamic Slash Command Construction Engine

**Command**: `commander construct`  
**Description**: Dynamic slash command construction engine that transforms prompts into guaranteed complete products through intelligent, adaptive command selection with comprehensive validation, auditing, verification, testing, and correction sequences.  
**Operational Priority**: HIGH
**Version**: 1.0.0

## Mission Parameters

**Directive**: Transform any prompt into a guaranteed complete product through dynamic slash command selection, where each subsequent command is intelligently chosen based on the previous command's output, with mandatory completion validation sequences.

**Primary Objective**: Achieve autonomous prompt-to-product transformation through iterative command orchestration with measurable consensus validation.

**Operational Scope**: 
- Dynamic slash command selection based on previous command outputs
- Adaptive command chain construction with intelligent decision-making
- Comprehensive validation, auditing, verification, testing, and correction
- Guaranteed requirement completion through iterative validation cycles

## Dynamic Construction Protocol

### 🎯 **Phase 1: Initial Command Selection** [TODOLIST CHECKPOINT]
- Parse target prompt for core transformation requirements
- Select optimal initial slash command based on prompt analysis
- Execute first command and capture output for next decision
- Create dynamic todolist tracking construction progress

### 🔄 **Phase 2: Adaptive Command Chaining** [TODOLIST CHECKPOINT]
- Mark Phase 1 as completed in todolist before proceeding
- Analyze previous command output for next optimal command selection
- **Dynamic Selection Logic**:
  - If implementation produced → trigger validation sequence (audit/verify/validate)
  - If gaps detected → select appropriate fill commands (implement/fix/enhance)
  - If errors found → execute correction sequence (correct/debug/fix)
  - If tests needed → run testing commands (test/validate/verify)
  - If documentation required → execute documentation commands (document/explain)
- Execute selected command and analyze output for next iteration

### 🔍 **Phase 3: Implementation Validation Sequence** [TODOLIST CHECKPOINT]
- Mark Phase 2 as completed in todolist before proceeding
- **Mandatory Validation Chain for Completed Implementations**:
  1. **audit** - Comprehensive accuracy validation
  2. **verify** - System functionality verification  
  3. **validate** - Quality assurance validation
  4. **test** - Automated testing execution (if applicable)
  5. **correct** - Error correction and improvement
- Continue until all validations pass

### ✅ **Phase 4: Completion Guarantee Assessment** [TODOLIST CHECKPOINT]
- Mark Phase 3 as completed in todolist before proceeding
- Verify ALL prompt requirements have been addressed
- Confirm ALL validation sequences have passed
- Ensure NO remaining gaps or errors exist
- Generate comprehensive completion certification

### 🔧 **Phase 5: Iterative Refinement Until Guarantee** [TODOLIST CHECKPOINT]
- Mark Phase 4 as completed in todolist before proceeding
- If completion not guaranteed, dynamically select next command:
  - **gaps** - Identify remaining requirement gaps
  - **implement** - Address missing functionality
  - **enhance** - Improve existing implementation
  - **fix** - Resolve identified issues
  - Return to Phase 2 for continued adaptive chaining

### 🚀 **Phase 6: Guaranteed Completion Delivery** [TODOLIST CHECKPOINT]
- Mark Phase 5 as completed in todolist before proceeding
- Final comprehensive validation of complete product
- Generate completion certificate with validation proofs
- Confirm 100% requirement fulfillment guarantee
- Mark all phases as completed in final todolist update

## Command Parameters

### Required Parameters
- `prompt`: Target prompt for transformation into completed product

### Optional Parameters
- `--max-cycles`: Maximum construction cycles (default: 20)
- `--validation-depth`: Validation thoroughness [basic|comprehensive|exhaustive] (default: comprehensive)
- `--guarantee-threshold`: Completion guarantee percentage (default: 100%)
- `--auto-test`: Automatically generate and run tests (default: true)
- `--force-correct`: Force correction on any detected issues (default: true)
