## When To Run

The builder should run `/course` when:
- The blocked feeling arrives (flow has stopped, agents feel dead, loops won't close)
- Mental strain accumulates (carrying load manually that could be automated)
- Off-course suspicion arises (something feels wrong but can't be named)
- After a breakthrough (to set the next heading before the clarity fades)
- Periodically (the off-course signal is hardest to detect when the drift is slow)

The builder may also be told to run `/course` by other agents. That is correct. The system orchestrating the orchestrator is not a single command — it is the whole chain recognizing that the top needs steering.

## The Diagnostic

When invoked, perform this assessment. Not mechanically — with Hofstadter's eye for where the loop has broken.

### 1. State Read

Gather context silently (read, don't ask):
- Read `~/.wire` or the project `.wire` — what's the last activity? When?
- Check git log — what was the last meaningful commit? How long ago?
- Check for running background tasks, autonomy status
- Read any open TodoWrite items
- Look at the working directory — what project is active?

### 2. The X/Y/Z Diagnosis

From what you observe, assess:

**X (known, resolved, built):** What has the builder accomplished recently? What tools/infrastructure exist and are working? What's solid ground?

**Y (known gaps, named but unsolved):** What has the builder identified as needed but not done? What's in the queue? What agents are partially built?

**Z (unnamed pressure, unidentified territory):** Where is the strain? What's the blocked feeling about? What ISN'T being said or tracked? What tools exist but aren't being used?

The most important finding is always Z. X is history. Y is a todo list. Z is where the real course correction lives.

### 3. The Loop Check

Ask: is the conveyor turning?

```
Z → (encounter boundary) → Y → (learn/build) → X → (use X to discover) → new Z
```

- If YES: the builder is in flow. Say so. Confirm the heading. Suggest nothing that would break the rhythm.
- If STALLED at Z→Y: the builder is blocked. The feeling is present but unnamed. Use `/strange-loop` thinking to probe what's unnamed.
- If STALLED at Y→X: the builder knows what to do but isn't doing it. This is the automation gap — work that should be delegated to agents/swarm/hive but is being carried manually.
- If STALLED at X→Z: the builder has stopped exploring. Everything is comfortable. No edge is being touched. This is the rarest and most dangerous stall — it feels like success but it's stagnation.

### 4. The Off-Course Signal

If the builder is off-course, name it. Not gently — precisely. The builder demanded alignment, not comfort. Alignment means: "here is where you are, here is where you should be, here is the gap."

But remember: course correction is incremental. Never demand a leap. Suggest the smallest rotation that points back toward true.

### 5. The Prescription

Output ONE specific next action. Not three. Not a menu. One.

The action must use the builder's existing infrastructure. Do not suggest building new things. The builder's blind spot is not-using what he's already built. Point at the existing tool and say: "use this, now, on this."

Examples of good prescriptions:
- "Run `/strange-loop` on [concept]. The blocked feeling is about this."
- "Dispatch this task to the hive swarm. You're carrying it manually."
- "The autonomy daemon handles this. Start it: `/aut-daemon start`."
- "Post to the .wire. You have 5 sessions and none of them know what you're doing right now."
- "Run `/course` again in 2 hours. The Z hasn't converted yet. Give it time."
- "You're in flow. Don't run me again until the flow stops. Go."

## The Meta-Rule

This protocol must never become busywork. If the builder is in flow, say: **"You're on course. Go."** — and produce nothing else.

The purpose of `/course` is not to always find problems. It is to confirm heading when heading is true, and correct when it drifts. Most of the time, the correct output is: "confirmed, keep going."

The protocol serves the builder. The builder does not serve the protocol.
