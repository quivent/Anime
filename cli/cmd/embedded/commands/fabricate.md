fabricate - Transform opus documentation suites into complete project implementations

Usage: Execute build-equivalent operations using opus command output as comprehensive schematic input for systematic project generation with architectural intelligence.

**Sequential Project Fabrication Protocol:**

🎯 **Phase 1: Documentation Suite Analysis & Validation**
- Parse opus documentation suite (README, PURPOSE, INTENT, CONCEPTS, METHODS, CLAUDE, SPECIFICATION)
- Validate cross-file consistency and specification completeness
- Extract technical requirements, architectural patterns, and implementation strategies
- Assess feasibility and identify technology stack requirements
- Generate implementation roadmap based on documentation analysis

🔍 **Phase 2: Implementation Strategy Selection & Architecture Design**
- Determine optimal programming language from SPECIFICATION.md technical requirements
- Select frameworks, libraries, and tools based on METHODS.md approaches
- Design project structure following CONCEPTS.md architecture principles
- Plan testing strategies and quality assurance from validation requirements
- Configure development environment and dependency management systems

🏗️ **Phase 3: Core Project Structure Generation**
- Create directory hierarchy per SPECIFICATION.md architecture diagrams
- Generate configuration files (package.json, requirements.txt, Cargo.toml, etc.)
- Initialize version control with appropriate .gitignore and repository setup
- Set up development environment configuration and build systems
- Create initial documentation structure and README synchronization

⚙️ **Phase 4: Feature Implementation & Business Logic**
- Implement core modules based on SPECIFICATION.md API specifications
- Develop business logic aligned with PURPOSE.md objectives and goals
- Create user interfaces following INTENT.md user experience requirements
- Add comprehensive input validation and error handling per METHODS.md protocols
- Integrate testing frameworks and initial test suite generation

🔧 **Phase 5: Integration & Enhancement Systems**
- Implement CLI interfaces and command-line tools if specified
- Add configuration management and environment variable systems
- Create deployment scripts, automation, and CI/CD pipeline setup
- Generate comprehensive documentation with auto-synchronization capabilities
- Set up monitoring, logging, and observability per specification requirements

✅ **Phase 6: Quality Assurance & Deployment Validation**
- Execute automated test suites and comprehensive validation frameworks
- Verify specification compliance across all implemented components
- Validate documentation accuracy and completeness against implementation
- Perform security auditing and performance validation testing
- Generate deployment readiness report and implementation summary

**Implementation Patterns:**

⚡ **Command Invocation**
```bash
fabricate ./ProjectDirectory
fabricate ./MyProject --strategy comprehensive --language python
fabricate ./WebApp --template-mode custom --testing-framework pytest
fabricate ./APIProject --deployment-target docker --architecture-pattern microservices
```

🔧 **Advanced Configuration Options**
- `--source-dir PATH`: Directory containing opus documentation suite (default: current directory)
- `--project-name NAME`: Override project name from documentation analysis
- `--implementation-strategy LEVEL`: Implementation depth (minimal|standard|comprehensive)
- `--language-preference LANG`: Programming language selection override
- `--template-mode MODE`: Template usage approach (existing|custom|hybrid)
- `--architecture-pattern PATTERN`: Override architectural decisions (monolith|microservices|serverless)
- `--dependency-management TOOL`: Package manager selection override
- `--testing-framework FRAMEWORK`: Test implementation approach selection
- `--deployment-target ENV`: Target deployment environment (local|docker|cloud|kubernetes)
- `--documentation-sync BOOL`: Maintain documentation synchronization with implementation
