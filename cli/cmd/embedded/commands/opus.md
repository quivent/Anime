opus - Project Genesis Documentation Suite Generator

Usage: Initialize new projects with comprehensive structured documentation based on application descriptions.

**Sequential Project Initialization Protocol:**

🎯 **Phase 1: Directory Validation & Setup**
- Validate target directory is empty or obtain user confirmation for existing files
- Create project workspace environment
- Initialize documentation structure framework
- Establish file creation permissions and access validation

🔍 **Phase 2: Description Analysis & Intent Extraction**
- Parse natural language application description for core themes
- Extract key concepts, objectives, and methodologies
- Identify target audience and use case scenarios
- Analyze technical complexity and scope requirements

📝 **Phase 3: Template Generation & Content Synthesis**
- Generate README.md with project overview, setup instructions, and usage examples
- Create PURPOSE.md defining core mission, objectives, and value propositions
- Develop INTENT.md outlining user goals, use cases, and success criteria
- Synthesize CONCEPTS.md explaining key terminology, principles, and frameworks
- Design METHODS.md detailing implementation approaches and methodologies
- Craft CLAUDE.md with AI collaboration guidelines and agent instructions
- Compose SPECIFICATION.md containing technical requirements and detailed specifications

🔧 **Phase 4: File Creation & Structure Implementation**
- Write all 7 documentation files with consistent formatting and professional structure
- Apply standardized markdown formatting and hierarchy
- Ensure cross-references and internal linking consistency
- Validate file integrity and content completeness

✅ **Phase 5: Validation & Quality Assurance**
- Verify all files created successfully with proper permissions
- Validate markdown syntax and structure compliance
- Check content coherence and completeness across all files
- Ensure documentation suite alignment with original description intent

📊 **Phase 6: Completion Report & Next Steps**
- Generate creation summary with file status and locations
- Provide recommended next steps for project development
- Display documentation suite overview and navigation guide
- Offer integration suggestions for development workflow

**Integration Patterns:**

⚡ **Command Invocation**
```bash
opus "Create a web application for task management with real-time collaboration features"
opus --dry-run "Mobile app for fitness tracking with social components"
opus --template custom "E-commerce platform with AI-driven recommendations"
```

🔧 **Advanced Options**
- `--dry-run`: Preview generated content structure without file creation
- `--template [name]`: Use custom template set for specialized project types
- `--verbose`: Display detailed generation process and content analysis
- `--interactive`: Step-through mode with approval gates for each file
