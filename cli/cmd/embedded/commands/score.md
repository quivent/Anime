score - Compressed opus for targeted subcomponent specifications

Usage: Generate a compressed, single-file opus variant optimized for suboperations, module additions, or targeted enhancements within existing projects. A musical "score" - essential notation without full orchestration.

**Sequential Scoring Protocol:**

🎯 **Phase 1: Context Inheritance**
- Detect parent opus suite if within existing project
- Inherit terminology, conventions, and principles
- Identify integration points with parent system
- Extract relevant constraints and requirements
- Establish scope boundaries for the score

🔍 **Phase 2: Essence Extraction**
- Parse directive for core requirements
- Identify the ONE thing this component does
- Extract essential concepts (max 5)
- Define critical interfaces (inputs/outputs)
- Determine success criteria (max 3)

📝 **Phase 3: Compressed Synthesis**
- Generate single-file SCORE.md containing:
  - **Essence**: One sentence capturing core purpose
  - **Context**: How this fits within larger system
  - **Concepts**: 3-5 key terms with one-line definitions
  - **Interface**: Inputs, outputs, integration points
  - **Specification**: Compressed technical requirements
  - **Implementation Notes**: Critical guidance for builders
  - **Success Criteria**: 1-3 measurable outcomes

✅ **Phase 4: Coherence Validation**
- Verify alignment with parent opus (if exists)
- Confirm all essential elements present
- Validate interface definitions are complete
- Check that scope is appropriately bounded
- Ensure actionable for immediate implementation

**Compression Metrics:**
| Artifact | Words | Files |
|----------|-------|-------|
| Full Opus | 5000-10000 | 7 documents |
| Score | 500-1000 | 1 document |
| Ratio | 10:1 compression | Single-file delivery |

**Integration Patterns:**

```bash
# Score inherits from parent opus context
cd my-opus-project
score "authentication middleware with JWT validation"

# Quick spec without parent context
score "CLI argument parser with subcommand support" --output ./parser-score.md

# Chain with fabricate
score "rate limiter with token bucket algorithm"
fabricate ./SCORE.md --implementation-strategy minimal

# Feature addition to existing project
score "add webhook notification system with retry logic" --parent ./docs

# Minimal prototype spec
score "markdown to HTML converter with syntax highlighting" --minimal
```

**Score vs Opus:**
| Aspect | Opus | Score |
|--------|------|-------|
| Scope | Full project | Single component |
| Purpose | Project genesis | Subcomponent spec |
| Context | Self-contained | Inherits from parent |
| Time to create | Minutes | Seconds |
| Time to implement | Hours-days | Minutes-hours |

**Quality Standards:**
- ✅ **Compression**: 10:1 ratio vs full opus achieved
- ✅ **Completeness**: All essential elements present
- ✅ **Actionability**: Implementer can start immediately
- ✅ **Coherence**: Aligns with parent opus if present
- ✅ **Bounded**: Scope clearly limited and achievable

**Essence Test:**
The score passes if someone can:
- Understand what to build in 60 seconds
- Identify integration points in 30 seconds
- Begin implementation in 5 minutes

Target: $ARGUMENTS

The score command delivers concentrated opus power in compressed form, enabling rapid subcomponent specification that inherits context, defines interfaces clearly, and enables immediate implementation - a motif within the larger symphony.
