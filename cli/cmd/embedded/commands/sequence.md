Execute comprehensive sequential procedure generation protocol for task decomposition, workflow creation, and systematic execution planning.

Usage: Generate systematic 8-phase sequential procedures for any task, file, folder, or CLI tool with autonomous execution coordination.

**Sequential Procedure Framework:**

🔍 **Phase 1: Input Analysis & Context Discovery**
- Input type identification (description/file/folder/CLI tool)
- Context analysis and requirement extraction
- Scope definition and boundary establishment
- Resource availability assessment and constraint identification

📋 **Phase 2: Task Decomposition & Objective Mapping**
- Primary objective identification and goal clarification
- Task breakdown into logical components
- Dependency analysis and prerequisite identification
- Success criteria definition and validation metrics

🎯 **Phase 3: Sequential Planning & Phase Design**
- 8-phase framework adaptation to specific task
- Phase transition criteria and checkpoint definition
- Parallel execution opportunities identification
- Risk assessment and mitigation planning

⚙️ **Phase 4: Tool & Resource Coordination**
- Required tool identification and availability verification
- CLI tool integration and command sequence planning
- Agent coordination and specialist assignment
- Environment setup and configuration requirements

🔄 **Phase 5: Execution Strategy & Workflow Design**
- Step-by-step procedure development
- Automation opportunities identification
- Quality gates and validation checkpoints
- Error handling and recovery procedures

📊 **Phase 6: Monitoring & Progress Tracking**
- Progress metrics definition and tracking setup
- Milestone identification and completion criteria
- Performance measurement and optimization opportunities
- Real-time monitoring and adjustment protocols

✅ **Phase 7: Validation & Quality Assurance**
- Output validation criteria and testing procedures
- Quality assessment and acceptance criteria
- Verification protocols and compliance checking
- Documentation and knowledge capture requirements

🚀 **Phase 8: Completion & Optimization**
- Final deliverable compilation and verification
- Process optimization and lessons learned
- Documentation generation and knowledge transfer
- Future iteration planning and improvement recommendations

**Input Processing Strategy:**

**📝 Description Processing:**
1. **Natural Language Analysis** - Parse requirements and intent
2. **Objective Extraction** - Identify core goals and outcomes
3. **Task Breakdown** - Decompose into actionable steps
4. **Sequence Generation** - Create systematic execution plan

**📁 File/Folder Processing:**
1. **Content Analysis** - Examine structure and contents
2. **Purpose Identification** - Determine intended operations
3. **Workflow Design** - Create appropriate processing sequence
4. **Tool Selection** - Choose optimal tools and methods

**🛠️ CLI Tool Processing:**
1. **Tool Verification** - Check accessibility and availability
2. **Capability Assessment** - Analyze features and options
3. **Usage Pattern Generation** - Create optimal usage sequence
4. **Integration Planning** - Design tool coordination workflow

**Sequence Configuration Options:**
- `--depth [basic|standard|comprehensive]`: Procedure detail level
- `--automation`: Emphasize automation and tool integration
- `--parallel`: Identify parallel execution opportunities
- `--validation`: Include comprehensive quality assurance
- `--interactive`: Generate interactive execution procedures

**CLI Tool Integration:**
```bash
# Single word CLI tool detection
if command -v $ARGUMENT >/dev/null 2>&1; then
    # Generate tool-specific usage sequence
    sequence_cli_tool "$ARGUMENT"
else
    # Process as description/path
    sequence_general "$ARGUMENT"
fi
```

Target input: $ARGUMENTS (description/file/folder/CLI tool)

The sequence protocol will analyze any input and generate systematic 8-phase execution procedures with tool coordination, automation opportunities, and quality assurance for optimal task completion.
